# Documentation Maintenance

Keep documentation aligned with the current behavior of the project.

## Update Docs After Behavior Changes

Update the affected documentation whenever behavior or workflows change.

Check the nearest relevant docs first, including repository guidance, command docs, and any affected skills.

Prefer small, direct edits that keep instructions accurate.
